12.2. Project Description#

Project aim and objectives

The aim of the project is to apply what you will learn in this module and think of ways to solve a robotic problem that requires the combination of computer vision and motion planning. You are required to write your own ROS2 code in python to let the robot perform the task autonomously in the simulation hosted by a simulator called Gazebo.

The objectives are as follows:

  • learn and apply the fundamental ROS2 framework for robotics systems

  • to apply motion planning methods

  • to understand and apply basic image processing techniques in autonomous robotic systems

The lab sessions are designed to fulfill these objectives as well as to provide you the necessary skills for the completion of the final ROS project.

Note

We offer the real turtlebot3 burger as well for you to play around and test your motion commands and image processing modules you have coded through the lab sessions. It is only formative, which is not accounted for your final grade of this module. From this week (as of 10/03/2025), a turtlebot will be brought to the usual lab sessions. Feel free to ask for it. The usage will be first-come-first-serve.

12.2.1. Specification and marking scheme (\(10\%\))#

  1. Your code runs with no problem (\(3\%\))

  2. Using what you learnt from the lectures, write a motion planner to command your robot (\(3\%\))

  3. Uploaded video to demonstrate a working solution, make sure the view is clear:

    • Robot has a live streaming of the camera feed with the detected objects (\(3\%\))

  4. Your robot finishes the task, and stays near the blue box (\(1\%\))
